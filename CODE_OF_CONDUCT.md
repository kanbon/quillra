# Code of Conduct

Quillra is a community of builders, contributors, reviewers, issue reporters, maintainers, and we all want working on it to feel safe and productive.

## The short version

Be decent. Be curious. Assume good faith.

- **Help people out.** Answer questions patiently, review PRs generously, explain when you disagree.
- **Criticise the code, not the person.** "This function has a race condition" is fine. Making it personal is not.
- **No harassment, personal attacks, or exclusionary behaviour.** Full stop.
- **Respect disagreement.** Reasonable people reach different conclusions. The maintainers make the call on contested decisions.
- **Keep it on-topic.** Issues and PRs are for Quillra; personal politics and off-topic flamewars belong somewhere else.

If you wouldn't say it in a professional Slack at a company you respect, don't say it here.

## The long version

We follow the **[Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)**: the community standard adopted by thousands of projects. The full text at that link describes expected behaviour, behaviour that is unacceptable, enforcement responsibilities, and the scope of this Code of Conduct.

That document is the authoritative reference for what is and isn't acceptable in our community spaces, GitHub issues, pull requests, discussions, code reviews, commit messages, and anywhere else Quillra is being discussed publicly.

## Reporting

If you experience or witness behaviour that violates this Code of Conduct, email the maintainers privately. Reports are treated confidentially. We will investigate, act proportionally, and follow up with the reporter when possible.

Maintainers can (and will) remove comments, close issues, revoke commit access, and ban contributors who repeatedly act in bad faith.
